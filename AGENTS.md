# Web studio operating rules

Use `skills/product-studio/` and `skills/add-reference/` as the active local studio system. Legacy `.agents/skills/` entries are retained temporarily for validated migration rollback only. Keep stable methodology in `knowledge/`, templates in `templates/`, validation in `scripts/`, and all mutable client/project state only in `<project-root>/.studio/`.

Read `knowledge/KNOWLEDGE_INDEX.md` and load only routed documents before project work. Preserve user data, back up material destructive rewrites, avoid duplicated facts, and never store client facts, screenshots, references, concepts, copy, or generated project documents in global plugin directories.

Use explicit concept approval before implementation. Build only approved active scope, verify the first complete journey and relevant quality gates, and report actual evidence and limitations. For a marketing hero, begin directly with H1 unless the user explicitly requested and approved a pre-heading; do not apply that website-specific rule to application screens.
