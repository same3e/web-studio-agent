# Reference analysis schema

Metadata schema version 1 requires stable ID, title, source type, original filename/URL, captured and added dates, dimensions when available, industries, platforms, style families, visual modifiers, primary/secondary roles, project types, sections, surfaces, interaction/responsive evidence status, status, confidence, hash/normalized URL where available, limitations, and no-copy constraints. Roles are industry, visual, product, structural, UX, interaction, content, and overlap.

Analysis separates observed evidence, interpretation, reusable principles, project use, strengths, weaknesses, risks, suitable/unsuitable projects, evidence locations, limitations, no-copy boundaries, and confidence. Static screenshots may not claim verified hover, sticky, responsive, animation, accessibility, or runtime behavior without inference/limitation markers.
