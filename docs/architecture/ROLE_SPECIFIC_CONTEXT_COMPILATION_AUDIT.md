# Role-specific context compilation audit

The baseline compiler accepted generic CLI state/task input, rebuilt maps each run, and selected globally required records. The v2 compiler derives its package from a ready assignment, canonical state records, a declarative role registry, selected rules, and applicable predecessor deltas. It writes an atomic manifest and canonical context index, estimates tokens as Unicode characters divided by four, and fingerprints the role package. Cache and staged host loading remain local structural behavior; native host reuse is unverified.
