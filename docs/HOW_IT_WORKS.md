# How it works

Web Studio Agent turns an idea into an approved and verified website or application. `product-studio` performs adaptive audit, requirements, stack recommendation, concepts, approval-gated implementation, and verification. `add-reference` writes project-local evidence in `.studio/references/`; product-studio reads that evidence without copying it into the plugin.
# Context-aware orchestration

`product-studio` selects lean, standard, or thorough context mode from active risk, creates a compact snapshot and repository map when needed, resolves routed rules, then compiles a role manifest before specialist work. Specialists return compact deltas for the next manifest. These packages select current pointers and evidence; they do not replace approval, factual, security, test, browser, or provenance gates.
