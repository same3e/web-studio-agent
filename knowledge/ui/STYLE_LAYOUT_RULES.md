# Style-specific layout interpretation

Styles are constraint sets, not generators. Apply the grid and quality rules in `LAYOUT.md` and `FRONTEND_QUALITY.md` first.

| ID | Style | Required structural interpretation |
| --- | --- | --- |
| SL-01 | Swiss / International | Explicit columns, baseline rhythm, limited scale, functional color, grounded asymmetry, restrained decoration; images align or deliberately break the grid. It is not merely Helvetica and left alignment. |
| SL-02 | Brutalism | Hidden or visible grid, readable type, controlled roughness and local chaos, intentional imbalance, accessible contrast, simplified mobile composition; negative offsets may not hide content. |
| SL-03 | Editorial | Text-led grid, deliberate reading measures, distinct headline/body/caption/metadata/pull-quote roles, connected captions, media-driven rhythm, and scale variation over excessive weight variation. |
| SL-04 | Minimalism | Exact alignment, deliberate spacing, strong type, purposeful components, and visible precision; emptiness may not simulate luxury or conceal missing content. |
| SL-05 | Maximalism | Base grid beneath primary, secondary, and ambient layers; controlled overlap, protected navigation/CTA legibility, fewer competing mobile layers, and motion hierarchy. |
| SL-06 | Bento / modular | Every module has a user/content function; size signals importance; hierarchy varies; long content fits; mobile avoids an endless sequence of identical cards. |
