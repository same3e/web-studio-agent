# Design rules

- Design for a commercial outcome and a clear primary conversion.
- Create a modern, distinctive direction; borrow principles from references, never their expression or assets.
- Use generous whitespace, strong hierarchy, intentional contrast, and purposeful imagery.
- Design mobile as its own composition, not a compressed desktop.
- Avoid scroll-jacking. Use motion only to clarify hierarchy, feedback, or state.
- Default stack: Next.js App Router, TypeScript, CSS Modules, Framer Motion, Vercel. Document project-specific changes.

## Hero hierarchy — mandatory default

Start every hero directly with its primary H1. The default order is: H1, explanatory text, primary CTA, and—only when useful—a short supporting note next to the CTA.

Do not add an eyebrow, overline, kicker, pretitle, pre-heading, hero label, category label, or any small decorative/uppercase line above H1 unless the user explicitly requested and separately approved that exact element. Never introduce one for hierarchy, whitespace, city, brand, category, editorial styling, or reference similarity. Put brand and location in the logo, navigation, body copy, or another appropriate element instead.
